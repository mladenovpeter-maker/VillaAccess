PROJECT OVERVIEW

Build a fully local-first AI-powered smart access control and vehicle recognition platform for vacation rental villas using Hikvision IP cameras, Linux server infrastructure, OCR, AI vehicle analytics, reservation logic, and gate/barrier control.

The goal is to completely replace HikCentral Professional with a custom modern platform.

The entire system should initially be WEB-BASED only.

All management must happen through a secure web dashboard with:

- login

- username/password authentication

- secure sessions/tokens

- role-based access later

- responsive mobile-friendly UI

In the future, the same backend APIs and business logic should support:

- React Native mobile app

or

- mobile wrappers

but for now EVERYTHING must work through the web dashboard.

====================================================

MAIN SYSTEM GOALS

====================================================

The system must:

- Detect vehicles entering/exiting the property

- Read license plates using OCR

- Perform vehicle analytics:

  - color recognition

  - make/model recognition

  - SUV/sedan/van classification

  - vehicle fingerprint embeddings

- Work FULLY OFFLINE locally

- Control gates/barriers using relay/alarm outputs

- Integrate with Hikvision IP cameras through:

  - RTSP

  - ISAPI

  - ONVIF if needed

- Integrate with Hikvision video intercom/access terminals

- Replace HikCentral completely

- Provide a modern web dashboard for administrators

====================================================

SYSTEM SCALE

====================================================

- 4 IP cameras total

- 4 Hikvision video intercom/access terminals

- 8 rental villas

- around 16 vehicles

- around 100-300 vehicle events per day

This is NOT a high-frequency traffic system.

The architecture should optimize for:

- event-based processing

- snapshots

- lightweight AI inference

NOT continuous heavy video analysis.

====================================================

OCR + AI REQUIREMENTS

====================================================

The system must NOT rely only on plate OCR.

It must use multi-factor vehicle identity:

- partial OCR

- vehicle make/model

- vehicle color

- vehicle type

- embeddings/fingerprints

- historical matching

Example:

Plate should be:

CB2020HR

But OCR only sees:

C?20?0?R

The AI should still recognize:

- Gray Mercedes GLE SUV

- Similar vehicle fingerprint

- Previous known visits

And decide to open the gate if confidence score is high enough.

====================================================

AI CONFIDENCE ENGINE

====================================================

The system should calculate trust/confidence scores.

Example logic:

>90% confidence:

- open automatically

70-90% confidence:

- ask admin confirmation

<70% confidence:

- deny access

====================================================

LOCAL AI REQUIREMENTS

====================================================

ALL AI processing must work LOCALLY.

The system must continue functioning even if internet is unavailable.

Do NOT use cloud AI APIs for realtime processing.

AI inference should run locally on the Linux server.

Use lightweight local models.
