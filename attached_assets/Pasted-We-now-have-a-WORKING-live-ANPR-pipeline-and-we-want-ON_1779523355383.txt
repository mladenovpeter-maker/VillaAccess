==================================================
THE MAIN PROBLEM TO SOLVE
==================================================

Current backend logic requires exact plate matching.

Example:

Database reservation:
CA3477MM

Worker OCR result:
CA3477M

Current result:
DENIED

This is WRONG behavior for a real ANPR system.

This should OPEN because:
- the OCR result is extremely close
- digits match correctly
- OCR confidence is acceptable
- only one trailing character is missing

We need SMART MATCHING instead of exact-only matching.

REQUIRED BACKEND IMPROVEMENTS

1. ADD FUZZY / PARTIAL PLATE MATCHING

If exact match fails:
- compare OCR plate with reservation plates
- calculate similarity score
- allow configurable threshold

Recommended:
Use Levenshtein similarity or SequenceMatcher ratio.

Examples:

OCR:

Reservation:
CA3477MM

This should produce HIGH similarity and should OPEN.
